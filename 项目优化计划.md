## 三、问题详细分析与解决方案

### 问题 1：JWT 密钥硬编码默认值

**位置：** `backend/src/main/resources/application.properties:17`

**当前代码：**
```properties
app.jwtSecret=${JWT_SECRET:DefaultSecretKeyForDevOnlyChangeInProduction2024}
```

**问题：** 生产环境若未设置环境变量将使用默认值，存在安全风险。

**解决方案：**
```properties
# 修改前
app.jwtSecret=${JWT_SECRET:DefaultSecretKeyForDevOnlyChangeInProduction2024}

# 修改后 - 强制要求环境变量
app.jwtSecret=${JWT_SECRET}
```

**验证步骤：**
1. 修改 `application.properties`
2. 重启后端服务
3. 验证未设置环境变量时启动失败

---

### 问题 2：数据库密码明文

**位置：** `backend/src/main/resources/application.properties:7`

**当前代码：**
```properties
spring.datasource.password=admin
```

**问题：** 数据库密码明文存储，存在安全风险。

**解决方案：**
```properties
# 修改前
spring.datasource.username=root
spring.datasource.password=admin

# 修改后
spring.datasource.username=${DB_USERNAME:root}
spring.datasource.password=${DB_PASSWORD}
```

**验证步骤：**
1. 修改 `application.properties`
2. 重启后端服务
3. 验证未设置环境变量时启动失败

---

### 问题 3：缺少登录失败限制

**位置：** `backend/src/main/java/com/bookstore/controller/AuthController.java`

**当前代码：**
```java
@PostMapping("/signin")
public ResponseEntity<?> authenticateUser(@Valid @RequestBody LoginRequest loginRequest) {
    // 直接认证，无失败次数限制
}
```

**问题：** 无登录失败限制，容易被暴力破解。

**解决方案：**
1. 创建 `LoginAttemptService` 记录登录失败次数
2. 超过阈值后锁定账户或延迟响应
3. 成功登录后重置失败次数

### 问题 4：密码强度验证不足

**位置：** `frontend/src/pages/Register.tsx`

**当前代码：**
```tsx
const validatePassword = (password: string) => {
    if (password.length < 6) {
        return '密码长度不能少于6个字符';
    }
    return null;
};
```

**问题：** 仅验证长度，无复杂度要求。

**解决方案：**
```tsx
const validatePassword = (password: string) => {
    if (password.length < 8) {
        return '密码长度不能少于8个字符';
    }
    if (!/[A-Z]/.test(password)) {
        return '密码必须包含大写字母';
    }
    if (!/[a-z]/.test(password)) {
        return '密码必须包含小写字母';
    }
    if (!/[0-9]/.test(password)) {
        return '密码必须包含数字';
    }
    return null;
};
```

### 问题 6：缺少 Service 接口

**位置：** 所有 Service 类

**问题：** Service 层直接使用实现类，不利于测试和扩展。

**解决方案：**
为每个 Service 创建接口：

```java
// BookService.java (接口)
public interface BookService {
    List<Book> getAllBooks();
    Page<Book> getAllBooks(Pageable pageable);
    Optional<Book> getBookById(Long id);
    // ...
}

// BookServiceImpl.java (实现)
@Service
public class BookServiceImpl implements BookService {
    @Autowired
    private BookRepository bookRepository;
    // ...
}
```

### 问题 7：批量操作无事务

**位置：** `OrderController.java:132-156`

**当前代码：**
```java
@PostMapping("/batch/status")
public ResponseEntity<?> batchUpdateOrderStatus(@RequestBody Map<String, Object> request) {
    // 缺少 @Transactional 注解
    int updatedCount = orderService.batchUpdateOrderStatus(orderIds, orderStatus);
}
```

**问题：** 批量操作无事务保证，可能导致数据不一致。

**解决方案：**
在 `OrderService.batchUpdateOrderStatus` 方法上添加 `@Transactional` 注解：

```java
@Transactional
public int batchUpdateOrderStatus(List<? extends Number> orderIds, OrderStatus status) {
    // ...
}
```

### 问题 8：LIKE 查询性能问题

**位置：** `BookRepository.java:20-24`

**当前代码：**
```java
@Query("SELECT b FROM Book b WHERE " +
       "LOWER(b.title) LIKE LOWER(CONCAT('%', :keyword, '%')) OR " +
       "LOWER(b.author) LIKE LOWER(CONCAT('%', :keyword, '%')) OR " +
       "LOWER(b.description) LIKE LOWER(CONCAT('%', :keyword, '%'))")
Page<Book> searchByKeyword(@Param("keyword") String keyword, Pageable pageable);
```

**问题：** LIKE 查询以 `%keyword%` 形式会导致全表扫描。

**解决方案：**
1. 添加数据库全文索引
2. 或集成 Elasticsearch
